## 개요

Eirene 메인넷은 테스트넷에서 검증된 안정적인 합의 알고리즘을 기반으로 구축됩니다. 메인넷 출시는 단계적으로 진행되며, 각 단계마다 안정성과 보안을 확보하기 위한 검증 과정을 거칩니다.

## 배포 일정

| 단계 | 일정 | 주요 활동 |
|------|------|-----------|
| 준비 단계 | D-90 ~ D-60 | 최종 코드 검토, 보안 감사, 성능 테스트 |
| 제네시스 준비 | D-60 ~ D-30 | 제네시스 파일 준비, 초기 검증자 선정 |
| 검증자 온보딩 | D-30 ~ D-15 | 검증자 설정 및 테스트 |
| 메인넷 출시 | D-Day | 제네시스 블록 생성 및 네트워크 활성화 |
| 안정화 단계 | D+1 ~ D+30 | 네트워크 모니터링 및 안정화 |
| 기능 활성화 | D+30 ~ | 거버넌스, IBC 등 고급 기능 단계적 활성화 |

## 네트워크 구성

### 초기 네트워크 매개변수

- 체인 ID: `zenanet-1`
- 블록 생성 시간: 4초
- 최대 검증자 수: 100
- 에포크 길이: 30,000 블록 (약 33시간)
- 최소 스테이킹 금액: 10,000 ZENA

### 합의 매개변수

```json
{
  "eirene": {
    "period": 4,
    "epoch": 30000,
    "slashingThreshold": 100,
    "slashingRate": 10,
    "missedBlockPenalty": 1
  }
}
```
