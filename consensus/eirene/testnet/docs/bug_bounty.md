# Eirene 테스트넷 버그 바운티 프로그램

## 프로그램 개요

Eirene 테스트넷 버그 바운티 프로그램은 다음과 같은 목표를 가지고 있습니다:

- Eirene 합의 알고리즘의 보안 취약점 및 버그 발견
- 네트워크 안정성 및 성능 향상
- 커뮤니티 참여 촉진
- 메인넷 출시 전 잠재적 문제 해결

프로그램 기간: 테스트넷 시작일부터 메인넷 출시 전까지

## 보상 범위

다음 영역에서 발견된 취약점 및 버그에 대해 보상을 제공합니다:

### 대상 범위

- **합의 알고리즘**: Eirene 합의 메커니즘의 취약점
- **네트워크 보안**: P2P 네트워크 관련 취약점
- **스테이킹 및 위임**: 스테이킹 및 위임 메커니즘의 취약점
- **거버넌스**: 거버넌스 시스템의 취약점
- **트랜잭션 처리**: 트랜잭션 처리 및 검증 관련 취약점
- **성능 이슈**: 심각한 성능 저하를 유발하는 문제

### 범위 제외

- 이미 알려진 이슈 (GitHub 이슈에 등록된 문제)
- 제3자 라이브러리의 취약점 (직접 보고해야 함)
- 사회 공학적 공격
- DoS 공격 (특별한 경우 제외)
- 문서 오류 또는 UI 관련 문제
- 로컬 환경에서만 발생하는 문제

## 보상 금액

취약점의 심각도에 따라 다음과 같은 보상을 제공합니다:

| 심각도 | 설명 | 보상 범위 |
|--------|------|-----------|
| 치명적 | 네트워크 중단, 자금 손실, 합의 실패 등 심각한 영향을 미치는 취약점 | $5,000 - $10,000 |
| 높음 | 네트워크 기능 일부 손상, 자금 위험, 성능 심각 저하 등 | $1,000 - $5,000 |
| 중간 | 특정 기능 손상, 제한된 영향을 미치는 취약점 | $500 - $1,000 |
| 낮음 | 최소한의 영향을 미치는 취약점 | $100 - $500 |

보상은 ZENA 토큰 또는 동등한 가치의 암호화폐로 지급됩니다.

## 보고 방법

취약점을 발견한 경우 다음 단계에 따라 보고해 주세요:

1. [버그 바운티 제출 양식](https://forms.gle/example-form-url)을 작성하세요.
2. 다음 정보를 포함해 주세요:
   - 취약점 제목 및 요약
   - 취약점 상세 설명
   - 재현 단계 (가능한 한 자세히)
   - 영향 평가
   - 완화 또는 수정 제안 (선택 사항)
   - 관련 스크린샷, 로그 또는 코드 (필요한 경우)

중요한 보안 취약점의 경우 공개적으로 공유하지 말고 보안 팀에 직접 이메일로 보내주세요: security@zenanetwork.com

## 평가 과정

취약점 보고서는 다음과 같은 과정으로 평가됩니다:

1. **접수 확인**: 보고서 접수 후 48시간 이내에 확인 이메일을 보냅니다.
2. **초기 평가**: 보안 팀이 보고서를 검토하고 유효성을 확인합니다 (1-3일).
3. **심각도 평가**: 취약점의 심각도를 평가합니다 (1-2일).
4. **검증**: 취약점을 재현하고 영향을 확인합니다 (1-7일).
5. **수정 개발**: 유효한 취약점에 대한 수정을 개발합니다.
6. **보상 결정**: 심각도에 따라 보상 금액을 결정합니다.
7. **보상 지급**: 취약점 수정 후 보상을 지급합니다.

전체 과정은 취약점의 복잡성에 따라 2주에서 4주가 소요될 수 있습니다.

## 규칙 및 조건

버그 바운티 프로그램 참여 시 다음 규칙을 준수해 주세요:

1. **책임감 있는 공개**: 취약점이 수정되기 전까지 공개하지 마세요.
2. **법률 준수**: 모든 관련 법률 및 규정을 준수하세요.
3. **선의의 행동**: 다른 사용자의 데이터에 접근하거나 서비스를 방해하지 마세요.
4. **중복 보고**: 이미 보고된 취약점은 보상 대상이 아닙니다 (선착순).
5. **품질**: 명확하고 자세한 보고서를 제출하세요.
6. **협력**: 필요한 경우 추가 정보를 제공하고 보안 팀과 협력하세요.
7. **비공개**: 취약점 세부 정보를 제3자와 공유하지 마세요.
